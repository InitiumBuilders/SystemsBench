## PROPOSAL (Step 4 — one enhancement)
- **Type:** new harness/defense layer — one coherent lever: *the elicit→parse bridge that makes the CLD + DYN
  deterministic scoring lanes runnable end-to-end against a model*. Additive, reversible, non-structural: it
  *implements* the already-governed §4.6 evaluation protocol (step 2 "fix the harness", step 4 "auto-grade
  deterministic formats") and the §3.1 structured schemas — it authors **no new rule, weight, scorer, oracle,
  or invariant**.
- **Target dimension:** none directly — the harness is **construct-neutral** (it changes how a response
  *reaches* the instrument, not what is measured). It serves the auto-graded lanes for **A** (CLD structure)
  and **D** (DYN trajectory) operationally. Does **not** alter the A–E composite weights.
- **Diff summary:** two staged artifacts, moved live at APPLY:
  - `engine/harness.py` — the bridge. Subcommands:
    - `template <CLD|DYN> <ITEM_ID>` → prints the item scenario + a canonical-schema instruction wrapper
      (with a NEUTRAL worked example that leaks no oracle answer) that elicits the exact structured fields the
      scorer reads.
    - `parse <CLD|DYN> <raw_response_file|->` → extracts the structured object from a raw reply
      (strip ```json fences → balanced-brace scan that respects strings/nesting → schema-validate), prints the
      clean `resp.json` to stdout. **Fails closed:** no JSON / not an object / missing a required key →
      `PARSE_ERROR — not scored: <reason>` on stderr, rc≠0. Never emits a fabricated or silent-empty resp.
    - `selftest` → fail-closed round-trip: for every oracle reference (CLD + DYN), serialize the canonical
      response in several wire-wrappings (bare · ```json fenced · prose-sandwiched · no-lang fence · brace in
      prose · DYN stringy-boolean+synonym), `parse` it back, and assert it scores **1.0** through the
      **existing** scorer; plus negatives (prose-only → PARSE_ERROR; missing-key → PARSE_ERROR) and a
      `template` render check. rc=0 iff every check passes (mirrors `cld-score.py`/`dyn-score.py` `calibrate`).
  - `items/harness_prompts.json` — machine-readable per-item scenario prompts (copied verbatim from the seed
    `.md` files) + the per-format schema spec + neutral worked example. The `template` data source.
- **Reuse boundary (the honest scope):** the parser does **NOT** re-implement semantic normalization — it
  hands the raw values to the scorer, whose existing `norm`/synonym/`cbool` logic does case/space/punct +
  synonym + boolean coercion. The parser's value-add is *extraction from messy text* + *shape validation* +
  *fail-closed*. Semantic variable **aliasing** (model words → oracle-canonical names) is **NOT** built — it
  would leak the oracle/inflate scores; held open as a fork for August + Ember.

## THOUGHT PROCESS (the narrative — why this move, what we expect, what we're watching)
The binding *structural* constraint (§3.1(b) accept-rule hardening) is PARKED with August + Ember and is not a
worker-applicable lever, so the run was scoped to the dominant *additive* move. Re-sensing the state, the
benchmark's whole judge-independent value lives in three deterministic scorers (SF/CLD/DYN) — yet every one of
them presupposes a hand-built structured `resp.json`. Nothing converts a live model's actual reply into that
file, and no prompt elicits the schema. So the auto-graded lane, the benchmark's crown jewel, was not actually
runnable against a model: a scorer with no front-end is an island. That is the lowest *actionable* Design-Review
gap (Gradeability 3; Discrimination 2 is live-data-blocked and unreachable from inside a SenseRun), and it is
the operator-named lever (BACKLOG #5). It is also Meadows #6 — an information flow that lets a model's reasoning
reach the measurement instrument.

The discipline that mattered most here was *restraint about scope*. The backlog text mentions "variable
aliases" — and the tempting over-build is a map from a model's words ("fish stock") onto the oracle's canonical
names ("FishPopulation"). I deliberately did **not** build that: a hand alias map leaks the oracle and inflates
scores, corrupting the very measurement the harness is supposed to feed. That is a measurement-validity
question — arguably structural — so it is held open as a named fork for August + Ember, not silently resolved
(Anti-Collapse). The honest, worker-safe scope is exactly two things: strong *elicitation* (a schema-instruction
wrapper that asks for the canonical fields, with a neutral non-leaking example) and robust *extraction* (pull
the JSON out of fenced/prose-wrapped text and validate its shape), then hand the raw values to the scorers —
whose existing case/space/punct + synonym + boolean normalization already does the semantics. The parser does
not duplicate that logic; the DYN synonym+stringy-boolean self-test and the CLD reverse-edge/full-word-loop
smoke test both prove the division of labor holds (still 1.0).

The other load-bearing choice was carrying the fail-closed invariant into the *parser*. A parser that emits an
empty or partial resp on a malformed reply would let the scorer compute a deceptive 0 — a fabricated number in
disguise. So `parse` raises and prints `PARSE_ERROR — not scored` (rc 3) on no-JSON / not-an-object /
missing-required-key. A blank is honest; a silent 0 is not (ANTIPATTERNS #1/#2).

What we expect: the next time a live CLD/DYN run is authorized, it is `template → model → parse → existing
scorer`, end to end, no hand-assembly. What we're watching: real model prose will stress the parser in ways the
self-test's canonical wire-wrappings cannot — multiple JSON blocks in one reply, partial schemas, variable
synonyms the scorer's lists don't cover. The first live run is where the parser's real coverage (and the
aliasing fork's necessity) gets measured. This run certifies only the round-trip of canonical responses — not
behavior on real prose, which needs the live spend the rail forbids here.
