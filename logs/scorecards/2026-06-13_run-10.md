**Highest-leverage fix chosen:** **build the per-format response-parsing + prompt-template harness layer for
CLD + DYN (BACKLOG #5)** ‚Äî one new module `engine/harness.py` (subcommands `template` / `parse` / `selftest`)
plus one machine-readable prompt source `items/harness_prompts.json`. The `parse` path turns a raw model reply
(fenced/ prose-wrapped JSON) into the exact `resp.json` the existing scorers consume, validating shape and
**failing closed** (`PARSE_ERROR ‚Äî not scored`, never a fabricated/empty resp). The `template` path emits, per
item, the scenario prompt + a canonical-schema instruction wrapper that elicits the structured fields. A
fail-closed `selftest` round-trips every oracle reference through several wire-wrappings ‚Üí parse ‚Üí existing
scorer == 1.0, plus negative tests (no JSON / missing key ‚Üí PARSE_ERROR).

**Why this is the dominant feasible lever (leverage thinking on ourselves):**
- It closes the lowest *actionable* Design-Review gap (Gradeability 3): the three deterministic scorers are
  the benchmark's whole judge-independent value, and **none is runnable against a live model without this
  bridge.** The harness converts "scorer exists" into "scoring lane is end-to-end executable" ‚Äî Meadows #6
  (information flow into the instrument), the precondition for every future live CLD/DYN run.
- It is **additive + reversible + construct-neutral**: a new module + data file, no change to any scorer,
  oracle, rubric, weight, or ¬ß1 invariant. Self-approvable; clean one-commit revert.
- It directly serves ANTIPATTERNS #1/#2 by carrying the fail-closed discipline into the *parser* itself: a
  response that cannot be parsed yields `PARSE_ERROR ‚Äî not scored`, never a silent 0 or an imputed number.
- It is the operator-named lever and a standing #8/#9 next-run seed.

**Held-open fork (NOT averaged away, NOT self-applied):** semantic *variable aliasing* for CLD ("fish stock"
‚Üî "FishPopulation"). A hand-maintained alias map from a model's words onto oracle-canonical names would
**leak the oracle and inflate scores** ‚Äî it is a measurement-validity question, arguably structural. This run
deliberately does NOT build it; the honest scope is strong *elicitation* (the template asks for the canonical
schema) + the scorers' existing case/space/punct + synonym normalization. The aliasing question is logged as
a fork for August + Ember (RECURSE), not silently resolved.
