Discrimination (cat 5) cannot be fixed this run — it needs response data from a live-model run, which a
SenseRun may not spend (§4.6 / rail), and it is downstream of having items at all. **Coverage (cat 6) is the
dominant *feasible* weakness** and a precondition for discrimination. Within coverage, the dominant format is
**CLD (Causal Loop Mapping)**:

1. **Largest construct hole.** CLD tests feedback-structure mapping (variables, signed links, R/B loops,
   delays, dominant loop) — Dimension A's core — which **no live format currently measures**. Biggest
   construct-validity gain per item.
2. **Opens a SECOND judge-independent scoring path TODAY** (mitigates ANTIPATTERNS #2, "single path to a
   number"). CLD's structural component (link polarity, loop sign = product of edge polarities, dominant loop)
   is deterministically checkable now — like SF — with no jury. The judge-for-completeness portion ships
   `UNCALIBRATED — not scored` (fail-closed) until CLD gold exists.
3. **Unblocks the highest-leverage additive idea (BACKLOG #2, programmatic CLD scorer, Meadows #6).** That
   lever is infeasible today — zero CLD items to score or validate. Seeding CLD turns it from "no inputs" to
   "ready next run." Strong compounding.

**Fix target:** seed CLD from 0 → §3.3 go-live threshold: **5 L3 items across 5 domains, each with a full
reference solution whose structural elements are deterministically gradeable.** Target dimension: **A** (with
B/D touched by behavior-over-time and dominant-loop reasoning).
