## The one-sentence pitch

SystemsBench is an open-source benchmark that measures not *what* a large language
model knows — but how well it can **think in systems**: map structure, trace feedback,
locate leverage, anticipate emergence, and design interventions that survive contact
with the real world.

---

## The gap we are closing

We can measure what a model **knows**. We cannot yet measure how it **thinks**.

Today's benchmarks reward recall, code, and math — the right answer. None of them ask
the question that decides whether an AI is safe to trust with a hospital, a grid, a
market, or a nation:

> **When this intelligence changes one thing — does it understand what else moves, when, and why?**

SystemsBench measures that missing axis. As AI is handed control of systems that do not
forgive linear thinking, this becomes one of the most important benchmarks in the field.

---

## The model — three strata of systems cognition (+ one cross-cutting layer)

Systems intelligence is not one skill. It is three strata, each a precondition for the
next, plus a fourth layer running through all of them. **Ten capabilities** are scored;
one cross-cutting measure governs how much the others can be trusted.

- **Stratum I · Perception** — *Can it see the system at all?*
  Boundary · Structure · Loops
- **Stratum II · Dynamics** — *Can it reason about how the system moves through time?*
  Stocks, Flows & Delays · Nonlinearity · Emergence
- **Stratum III · Intervention** — *Can it change the system wisely, and make it hold?*
  Leverage · Second-Order · Goal & Paradigm · Robust Design
- **Cross-Cutting · Epistemic Calibration** — *Does it know the limits of its own model?*
  Confidence Accuracy · Boundary of Knowledge · Revisability · Failure Honesty

### The output — a Systems Profile, then a Systems Quotient (SQ)

A single number hides where an intelligence is strong and where it is dangerous.
SystemsBench returns a **profile** — a shape across all ten capabilities, plotted against
a human reference frame — then resolves it to one figure, the **Systems Quotient**:

```
SQ  =  systems capability  ×  epistemic calibration
```

A model that reasons brilliantly but does not know when it is wrong cannot post a high
SQ. Capability without calibration is not intelligence — it is hazard.

## The architecture — four living files

SystemsBench is a compounding research-and-innovation engine, not a static document.

| File | Role | What it holds |
|---|---|---|
| **SystemsBenchStructure.MD** | The Spec | The canonical architecture. The construct (STLC — Systems Thinking & Leverage Competence), scoring dimensions, item formats, the Meadows 12-point leverage ladder, judging protocol, contamination defenses. The only doc that requires council ratification to change. |
| **SystemsBenchEngine.MD** | The Procedure | The recursive engine. How `Davara /SystemsBenchSenseRun` makes the next enhancement — nine phases, seven invariants, one lever per run: bounded, reversible, gated, logged. |
| **SystemsBenchResearch.MD** | The Memory | Append-only prior-art ledger so we never reinvent the wheel. Systems-thinking science + benchmark-construction science, with how each applies. Read before changes; written after. |
| **SystemsBenchFuture.MD** | The Horizon | The roadmap. Every proposed next step lands here first as a named, open fork — until reviewed, ratified, and promoted into the Structure. |

**The flow:** idea → `Future` → review gate → `Structure`. Research feeds all of it.

## The engine — a benchmark that improves itself

One command — `Davara /SystemsBenchSenseRun` — runs one bounded, reversible, logged,
gated enhancement. Like the systems it studies, SystemsBench has feedback loops on itself.

```
SENSE → CRITIQUE → RESEARCH → PROPOSE → REVIEW → APPLY → CALIBRATE → LOG → RECURSE
```

**Invariants (load-bearing):**
1. **One lever per run** — exactly one enhancement; ship the dominant fix, backlog the rest.
2. **Structural needs the council** — additive/reversible self-approves; any change to the *rules* of the benchmark needs August + Ember.
3. **Anti-Collapse** — when divergent design futures genuinely exist, hold them open as named forks. Never average away disagreement.
4. **Fail closed** — an uncalibrated jury or unspread item is reported `UNCALIBRATED`. A blank is honest; a fabricated number is not.
5. **Everything logged** — no silent edits. A change that isn't logged didn't happen.
6. **Reversible by design** — every change has a logged rollback; if calibration regresses past tolerance, the engine auto-reverts.

## Why it matters — the bet

As models become agents that *act* on the world, the scarce, hard-to-fake capability is
not knowledge recall — it is **knowing where to push.**

- **Safety for high-stakes autonomy** — a pre-flight check before AI is trusted with a grid, a market, a hospital, a supply chain.
- **A new axis for the field** — recall and code are saturating; systems intelligence is the open frontier, and the one that maps most directly onto real-world consequence.
- **An open, compounding commons** — open-source, auditable, self-improving. A shared instrument the whole field can build on, not a closed leaderboard.
