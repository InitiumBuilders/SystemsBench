## v0.8.1 — 2026-06-13 — scorer evolution — CLD v2: partition-robust topology+sign matching (the First-Light fix)
- **`engine/cld-score.py` rewritten to v2 — match CLD loops by TOPOLOGY + SIGN, not by node name.** First Light (`results/FIRST-LIGHT-2026-06-13.md`, Finding 4) caught the v1 ruler scoring a *structurally correct* answer at **0.04** purely because it renamed `OpenIncidents → OpenIncidentQueue` and collapsed two oracle nodes into one. v1 keyed loops on an **exact node-set** and edges on **exact names** — penalizing **vocabulary and partitioning choices**, not reasoning. v2 fixes the measurement-validity failure the two deepest Meadows lessons name directly: **"the behavior of a system cannot be known just by knowing the elements"** (score structure, not element-labels) and **"there are no separate systems … where to draw a boundary depends on the purpose"** (a different-but-valid partition is valid systems thinking).
- **Method:** a loop's **polarity = product of its signed edges** is invariant under *both* renaming *and* reasonable re-partitioning (collapsing a same-sign chain preserves the product), so polarity is the primary **name-free** key. Each oracle loop is matched to a response loop **GATED on equal polarity**, disambiguated by **soft node overlap**. Added an **internal-consistency** term (does each response loop's declared R/B label equal the product of *its own* edges? — a fully name-free coherence probe). Dominant-loop + dynamic-insight (shift / driving-delay) now ride on the soft correspondence, so they survive renaming. Soft name matching uses **generic fuzzy string similarity only** (char-trigram Dice + shared-stem containment) — **never an oracle-specific synonym dictionary** (the hand-alias-map fork stays CLOSED — it would leak the oracle; a §5 measurement-validity question for August + Ember).
- **Weights rebalanced toward structure** (Meadows #1): v1 `0.50·edges + 0.30·loops + 0.20·vars` → v2 `0.50·loops + 0.20·edges + 0.10·vars + 0.20·consistency`. Component weights remain **tool-internal policy** (tunable, calibrate-first), **NOT governance**. **v1 is a special case of v2** (exact names → soft-overlap 1.0) → the change can only *recover* name-strictness false-negatives, never lower an honest score.
- **Calibrate-first + no-regress (the gate):** **CLD calibrate 36/36 PASS** (oracle loop-signs 11/11; references round-trip 1.0; partial=0.5; trap≤0.25; omitted-loop<1.0; **+ a new per-item probe: a fully renamed-but-isomorphic answer still scores 1.0** — the topology+sign guarantee). **harness selftest 61/61**, **DYN calibrate 34/34** (untouched). **Zero regressions** asserted in code over all 10 First-Light responses.
- **Re-score evidence** (`results/CLD-V2-RESCORE-2026-06-13.md` + `results/cld-v2-rescore-2026-06-13.json`): CLD mean **Davara 0.30 → 0.81, Davaris 0.31 → 0.73**. Recovery is *principled*, not laundered — discrimination **improved**: full-dynamic-insight answers land 0.83–0.93; answers that got the loops but **missed the delay/shift insight cap cleanly at 0.50** (CLD-ORG-003, CLD-MKT-005) — that 0.50 is now reasoning-signal, not vocabulary-noise. Traps still hold (0 fired). Resolves the First-Light "CLD is a lower bound" caveat — *not* by an oracle-leaking alias map, but by reading the structure the model drew.
- **Scope:** deterministic-scorer evolution (§4.6 step-4 territory). **The CLD completeness/quality (jury) sub-score is UNCHANGED — still `UNCALIBRATED — not scored`** (no CLD gold; fail-closed §5.1); no jury number emitted. **The PARKED §3.1(b) accept-rule hardening (Meadows #5) was NOT touched** — it remains with August + Ember. Additive + reversible.

## v0.8.0 — 2026-06-13 — SenseRun #10 — harness layer (CLD+DYN response-parsing + prompt-template bridge; auto-graded lanes now runnable end-to-end)
- **`engine/harness.py`** — built **BACKLOG #5, the per-format response-parsing + prompt-template harness layer** for CLD + DYN (Meadows #6 information-flow into the instrument). The three deterministic scorers (`SF`/`CLD`/`DYN`) consume a *structured* `resp.json`, but nothing turned a live model's raw reply into that file and no prompt elicited the canonical schema — the auto-graded lane was not runnable against a model. Subcommands: `template <CLD|DYN> <ITEM_ID>` emits the item scenario + a canonical-schema instruction wrapper (with a NEUTRAL worked example that leaks no oracle answer); `parse <CLD|DYN> <FILE|->` extracts the structured object from a raw reply (strip ```json fences → balanced-brace scan respecting strings/nesting → schema-validate) and prints the exact `resp.json` the scorer reads; `selftest` is the fail-closed round-trip.
- **Implements the already-governed §4.6 harness step + §3.1 structured schemas** — authors **no new rule, weight, scorer, oracle, item, or invariant**. **Construct-neutral:** it changes how a response *reaches* the instrument, not what is measured.
- **REUSE not reinvention:** the parser does **NOT** re-implement semantic normalization — it hands raw values to the scorers, whose existing `norm`/synonym/`cbool` logic does case/space/punct + synonym + boolean coercion. The parser's value-add is extraction-from-messy-text + shape-validation + **fail-closed** (`PARSE_ERROR — not scored` on no-JSON / not-an-object / missing-required-key — never a fabricated or silent-zero resp; ANTIPATTERNS #1/#2 carried into the parser).
- **`items/harness_prompts.json`** — machine-readable per-item scenario prompts (copied verbatim from the seed `.md` files) + the per-format schema spec + neutral worked example; the `template` data source. (`HARNESS_PROMPTS` env var overrides the path for tests/CI.)
- **Fail-closed self-test** (`selftest`, mirrors the scorers' `calibrate`): every CLD+DYN oracle reference, serialized in five wire-wrappings (bare · ```json fenced · prose-sandwiched · no-lang fence · brace-in-prose), parses back and scores **1.0** through the **existing** scorer; DYN synonym+stringy-boolean replies also → 1.0 (proving the scorer's normalization, not the parser, does semantics); prose-only and missing-key → PARSE_ERROR; templates render — **61/61 PASS**, rc=0. End-to-end smoke: a prose-wrapped DYN naive reply → `trajectory_subscore 0.1, trap_fired`; a CLD reply with reverse-direction edges + full-word loop type → `1.0`; an unparseable reply → `PARSE_ERROR`, rc 3. No live-model spend (§4.6/rail).
- **Effect:** the CLD + DYN deterministic scoring lanes are now **runnable end-to-end against a live model** (elicit → parse → existing scorer) — the robustness precondition for any live CLD/DYN run. No measurement/construct change; CLD/DYN jury sub-scores stay `UNCALIBRATED — not scored`; no jury number emitted; no IRT measured.
